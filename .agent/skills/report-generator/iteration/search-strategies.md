# 搜索深度策略配置

## 使用说明

本文档定义四级资料搜集深度的详细策略、来源优先级、验证标准和时间预算。

## 搜索深度级别

### Level 1: 快速搜索 🔍

#### 概述
适用于基础概念理解、通用知识查询等不需要深入研究的场景。

#### 配置参数
| 参数 | 值 |
|-----|-----|
| 资料数量 | 1-2个 |
| 时间预算 | 3-5分钟 |
| 来源优先级 | 百度百科 → 维基百科 → 1个权威官网 |
| 验证标准 | 来源可信赖、信息一致 |
| 缓存策略 | 优先使用缓存，缓存有效期60天 |

#### 来源优先级
1. **百度百科** - https://baike.baidu.com/
   - 用途：基础概念、定义解释
   - 特点：中文内容丰富、更新较快
   - 验证：查看编辑历史、参考资料

2. **维基百科** - https://zh.wikipedia.org/
   - 用途：国际概念、多语言对照
   - 特点：内容权威、引用规范
   - 验证：查看引用文献、编辑讨论

3. **权威官网** - 1个
   - 用途：官方定义、权威数据
   - 示例：国家标准全文公开系统、行业协会官网
   - 验证：确认网站性质（.gov、.org等）

#### 输出格式示例
```markdown
## 快速搜索结果：[主题]

**核心定义**：
[从百度百科或维基百科提取的简明定义]

**关键要点**：
- 要点1
- 要点2
- 要点3

**数据参考**：
- 数据1 (来源: XXX)
- 数据2 (来源: YYY)

**来源**：
1. 百度百科 - [URL]
2. 维基百科 - [URL]

**验证结果**：✅ 信息一致，来源可信
```

#### 适用场景
- [x] 基础概念查询（如"什么是生态修复"）
- [x] 通用原理说明（如"风力发电原理"）
- [x] 快速信息收集（如"项目背景"章节）
- [x] 辅助信息补充（如"备注"、"说明"）

### Level 2: 标准搜索 📚

#### 概述
适用于技术方案设计、数据分析等需要一定深度的场景。

#### 配置参数
| 参数 | 值 |
|-----|-----|
| 资料数量 | 3-5个 |
| 时间预算 | 8-10分钟 |
| 来源优先级 | 国家标准/行业标准 → 官方技术白皮书 → 行业报告 → 学术论文摘要 |
| 验证标准 | 至少2个来源一致、数据有时间标注 |
| 缓存策略 | 优先使用缓存，缓存有效期45天 |

#### 来源优先级
1. **国家标准/行业标准**（GB/T、NB/T、DL/T等）
   - 来源：国家标准全文公开系统、行业标准信息平台
   - 示例：
     - GB/T XXXXX-202X 风力发电机组技术要求
     - NB/T XXXXX-2022 风力发电场设计规范
   - 验证：确认标准编号、发布日期、适用范围

2. **官方技术白皮书**
   - 来源：政府机构、行业协会、大型企业
   - 示例：
     - 国家电网《风电场建设技术白皮书》
     - XX省生态环境厅《生态修复技术导则》
   - 验证：确认发布机构、发布时间

3. **行业报告**
   - 来源：咨询公司、研究机构、媒体报告
   - 示例：
     - 艾瑞咨询《2024年中国风电行业发展报告》
     - 智研咨询《生态修复市场分析报告》
   - 验证：确认发布机构、报告时间、数据来源

4. **学术论文摘要**
   - 来源：知网、万方、维普
   - 示例：
     - 《黄河流域生态修复技术研究》（环境科学学报）
     - 《风电场环境影响评估方法研究》（生态学报）
   - 验证：确认期刊质量、发表时间、作者背景

#### 输出格式示例
```markdown

### 标准依据

  - 发布日期: 2023-06-15
  - 核心要求: [关键要求1、2、3]
  - 适用范围: [适用范围说明]

### 技术方案参考
- [技术白皮书或导则]
  - 发布机构: XXX
  - 发布时间: YYYY-MM-DD
  - 关键技术: [关键技术列表]

### 行业数据
- [行业报告]
  - 发布机构: XXX
  - 发布时间: YYYY-MM-DD
  - 核心数据: [关键数据]

### 学术研究
- [学术论文]
  - 作者: XXX
  - 发表时间: YYYY-MM
  - 核心结论: [研究结论]

**来源验证**：✅ 3个来源一致，信息可信
**置信度评分**：⭐⭐⭐⭐ (4/5)
```

#### 适用场景
- [x] 技术方案编写（如"技术方案概述"章节）
- [x] 数据分析（如"现状分析"、"数据统计"）
- [x] 规范标准引用（如"设计依据"、"技术要求"）
- [x] 核心技术描述（如"关键技术"、"技术路线"）

### Level 3: 深度搜索 🔬

#### 概述
适用于核心技术、创新方案等需要深入研究、多源验证的场景。

#### 配置参数
| 参数 | 值 |
|-----|-----|
| 资料数量 | 6-10个 |
| 时间预算 | 15-20分钟 |
| 来源优先级 | 标准全文 + 专利文档 + 学术论文 + 政府报告 + 企业研究 |
| 验证标准 | 至少3个来源交叉验证、研究方法透明 |
| 缓存策略 | 优先使用缓存，缓存有效期30天 |

#### 来源优先级
1. **标准全文**（2-3个）
   - 来源：国家标准全文公开系统、行业数据库
   - 示例：
     - GB/T XXXXX-202X 风力发电机组技术要求（全文）
     - GB/T XXXXX-202X 生态修复技术规范（全文）
   - 验证：逐条核对技术参数、适用范围

2. **专利文档**（2个）
   - 来源：中国专利公布公告、美国专利局
   - 示例：
     - CNXXXXXXXXXXXX.X 河道生态修复方法
     - CNXXXXXXXXXXXX.X 风力发电机组优化装置
   - 验证：确认专利状态、发明点、技术方案

3. **学术论文**（2-3个）
   - 来源：知网、万方、维普、Web of Science
   - 示例：
     - 《黄河流域生态修复技术研究》（环境科学学报，2023）
     - 《低风速风力发电机组优化设计》（可再生能源，2022）
   - 验证：确认期刊级别、影响因子、引用次数

4. **政府报告**（1-2个）
   - 来源：国务院、部委、地方政府
   - 示例：
     - 《黄河流域生态保护和高质量发展规划纲要》
     - 《XX省新能源发展规划（2021-2030）》
   - 验证：确认发文单位、文件号、时效性

5. **企业研究**（1个）
   - 来源：大型企业、研究机构
   - 示例：
     - XX风电公司《低风速技术研究报告》
     - XX环保科技《生态修复技术白皮书》
   - 验证：确认企业背景、研究实力

## 验证标准

### 三级验证流水线

#### 1. 格式/元数据检查（自动化）
- [ ] DOI/URL有效性
- [ ] 书目元数据完整性（标题、作者、日期、来源）
- [ ] 时间戳合理性（不能是未来时间）

#### 2. 书目标验（自动化）
- [ ] 对比权威数据库（知网、国家标准全文公开系统）
- [ ] 检查出版物合法性（是否真实存在）
- [ ] 验证作者归属（作者是否真实）

#### 3. 语义验证（AI辅助）
- [ ] 语义相似度分析（多个来源对比）
- [ ] 矛盾检测（不同来源是否矛盾）
- [ ] 支持度分类（支持/部分支持/不支持/不确定）

## 多源协同验证

### 独立源检索
- 跨不同来源搜索相同主张
- 避免依赖单一信息源
- 确保信息来源多样性

### 多数投票
- 聚合多个来源的裁决
- 以多数观点为准
- 少数观点作为补充说明

### 加权共识
- 优先考虑权威来源（同行评审期刊、.gov域名）
- 国际来源权重高于国内来源
- 近期来源权重高于旧来源

### 时间分析
- 优先使用近3年资料
- 过期资料标注原因
- 定期更新旧资料

## 冲突解决策略

### 并列呈现
- 冲突来源并排展示
- 说明各方的观点
- 不偏向任何一方

### 可信度指示
- 包含来源可信度指标
- 说明各来源的权威性
- 帮助用户判断

### 差异解释
- 提供差异的上下文解释
- 分析差异产生的原因
- 给出建议意见

### 不确定性标记
- 显著分歧时标记"不确定"
- 说明不确定性来源
- 建议进一步验证

## 置信度评分算法

```python
confidence = (
    source_reliability_weight * domain_authority +
    evidence_alignment_weight * semantic_similarity +
    cross_reference_weight * source_corroboration +
    freshness_weight * publication_recency
)
```

### 评分说明
- **source_reliability_weight** (0.35)：来源可靠度（权威机构、同行评审）
- **evidence_alignment_weight** (0.25)：证据一致性（多个来源对比）
- **cross_reference_weight** (0.25)：交叉引用（至少2个来源支持）
- **freshness_weight** (0.15)：时效性（近3年优先）

### 评分等级
- ⭐⭐⭐⭐⭐ (5/5)：非常可信，可直接使用
- ⭐⭐⭐⭐ (4/5)：可信，可参考使用
- ⭐⭐⭐ (3/5)：基本可信，需要验证
- ⭐⭐ (2/5)：可信度低，需要多方验证
- ⭐ (1/5)：不可信，不建议使用
