## 反馈记录规范

### 记录时机
- 每章确认后立即记录章节反馈
- 报告完成后记录整体反馈
- 定期（每周）汇总分析反馈

### 记录内容
- **章节反馈**：章节编号、满意度、修改次数、修改类型、具体意见
- **整体反馈**：整体满意度、总耗时、总修改次数、用户评价、改进建议

### 反馈类型定义

#### 修改类型
1. **术语统一**：术语使用不一致，需要统一表述
2. **内容补充**：内容不够完整，需要补充信息
3. **格式调整**：格式需要调整，如标题层级、段落长度
4. **图表调整**：图表需要调整，如类型、样式、数据
5. **逻辑修改**：逻辑结构需要调整，如章节顺序、内容组织

#### 问题类型
1. **术语不一致**：章节间术语表述不统一
2. **数据引用缺失**：数据缺少来源引用
3. **逻辑不连贯**：章节间逻辑衔接不自然
4. **内容冗余**：内容重复或冗余
5. **图表不清晰**：图表表达不清晰或不准确

## 反馈驱动改进

### 改进流程
```
收集反馈 → 分析问题 → 识别模式 → 生成建议 → 用户确认 → 应用改进
    ↓          ↓          ↓          ↓          ↓          ↓
 记录日志   统计分析   找出共性   评估效果   用户批准   验证效果
```

### 改进优先级
- **高优先级**：影响用户满意度、出现频率高的问题
- **中优先级**：影响用户体验、出现频率中等的问题
- **低优先级**：改进用户体验、出现频率低的问题
