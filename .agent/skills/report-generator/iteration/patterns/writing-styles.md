# 写作风格模式

## 使用说明

本文档积累不同用户的写作风格偏好，实现个性化适配和风格一致性。

## 风格分类

### 风格类型

| 风格ID | 风格类型 | 使用用户 | 出现次数 | 特征描述 | 适用场景 |
|--------|---------|---------|---------|---------|---------|
| S001 | 正式严谨型 | user001 | 8次 | 语言规范、术语准确、结构完整 | 学术报告、技术方案 |
| S002 | 简洁明了型 | user002 | 6次 | 段落简短、重点突出、逻辑清晰 | 工程报告、施工方案 |
| S003 | 详实丰富型 | user003 | 5次 | 内容详实、案例丰富、数据充实 | 可行性报告、立项报告 |
| S004 | 图文并茂型 | user004 | 4次 | 图表丰富、可视化强、直观易懂 | 汇报材料、演示文档 |

### 风格特征详细

#### 风格001：正式严谨型

**语言特征**：
- [x] 使用规范的工程技术语言
- [x] 术语使用准确、统一
- [x] 避免口语化表达
- [x] 句式结构完整、规范

**术语使用**：
- [x] 首次出现给出完整定义
- [x] 重要术语中英文对照
- [x] 严格统一术语使用
- [x] 避免同义词混用

**数据使用**：
- [x] 数据精确、有据可查
- [x] 数据来源标注清楚
- [x] 数据使用规范（单位、格式）
- [x] 避免模糊数据（"约"、"左右"）

**段落长度**：
- 段落长度：中等（3-5行）
- 每段主题：单一明确
- 段落过渡：使用连接词（"此外"、"另外"、"因此"）

**逻辑结构**：
- 章节结构：层次分明
- 论证方式：归纳+演绎
- 逻辑衔接：严谨自然
- 结论推导：充分可信

**用户反馈**：
- "语言专业、规范"（6次）
- "术语使用准确"（5次）
- "结构严谨、逻辑清晰"（4次）

**适用章节**：
- 技术方案、工程设计、质量保证

**适用用户**：
- 专业技术评审、学术研究人员

---

**语言特征**：
- [x] 语言简洁、直截了当

- [x] 避免冗余表达
- [x] 使用短句、易于理解

**术语使用**：
- [x] 术语定义简明扼要
- [x] 避免过度使用专业术语
- [x] 必要时提供通俗解释
- [x] 术语选择基础、常用

**数据使用**：
- [x] 数据直观、突出重点
- [x] 使用表格对比数据
- [x] 关键数据加粗标注
- [x] 使用近似值（"约"、"左右"可接受）

**段落长度**：
- 段落长度：短小（1-2行）
- 每段主题：单一明确
- 段落过渡：简短直接（"总之"、"因此"）

**逻辑结构**：
- 章节结构：简单明了
- 论证方式：直接论证

**用户反馈**：

**适用章节**：

**适用用户**：

**语言特征**：
- [x] 语言详实、内容丰富
- [x] 案例引用、实例说明
- [x] 多角度阐述、全面分析

**术语使用**：
- [x] 术语定义详细、充分
- [x] 提供术语的历史和背景
- [x] 使用多种术语表达同一概念
- [x] 术语使用灵活、多样

**数据使用**：
- [x] 数据丰富、多维度
- [x] 提供历史数据对比
- [x] 使用区间数据（"范围"、"区间"）
- [x] 数据来源多样化

**段落长度**：
- 段落长度：较长（5-8行）
- 每段主题：可包含多个子点
- 段落过渡：详细过渡（"具体来说"、"举个例子"）

**逻辑结构**：
- 章节结构：层次丰富
- 论证方式：多角度论证
- 逻辑衔接：详细展开

**用户反馈**：

**适用章节**：

**适用用户**：
- 决策领导、投资方、政府部门

**数据使用**：

## 风格匹配

### 匹配流程
1. 收集用户反馈和历史记录
2. 分析用户偏好（语言、术语、数据、段落）
3. 计算风格匹配度
4. 选择最匹配的风格
5. 在章节编写时应用该风格

### 匹配度计算
```
匹配度 = (
    语言匹配度 * 0.3 +
    术语匹配度 * 0.3 +
    数据匹配度 * 0.2 +
    段落匹配度 * 0.2
)
```

### 匹配阈值
- **≥ 80%**：高度匹配，直接应用
- **60% - 79%**：基本匹配，可应用微调
- **< 60%**：不匹配，询问用户或使用默认风格

### 风格切换
- 每个报告开始时询问风格偏好
- 允许章节间切换风格（需用户确认）
- 记录风格切换历史

## 风格库管理

### 添加流程
1. 识别新的写作风格
2. 记录风格特征（语言、术语、数据、段落、逻辑）
3. 收集用户反馈
4. 分析风格适用场景
5. 评估风格质量（使用次数≥3次，满意度≥4星）
6. 添加到本风格库

### 更新流程
1. 定期（每完成10次使用后）分析风格使用情况
2. 识别需要优化的风格
3. 收集用户最新反馈
4. 更新风格特征和适用场景
5. 保留历史版本

### 推荐到共享库条件
推荐到 iteration/shared/writing-styles-shared.md 的条件：
- 使用次数 ≥ 5次
- 用户满意度 ≥ 4星
- 风格特征清晰、可复用
- 经过至少2次用户反馈和优化

## 风格应用模板

### 正式严谨型应用模板
```markdown
## [章节标题]

### [小节标题]
[本节内容]

根据《[标准名称]》（GB/T XXXXX-202X）的要求，[技术要求1]。[技术要求2]。[技术要求3]。

[术语1]是指[术语定义]。在[具体场景]中，[术语1]的应用尤为重要。

数据表明，[数据1]（来源：XXX）。此外，[数据2]（来源：YYY）。综合分析可知，[结论]。

因此，本[章节/项目]的设计符合[标准/规范]的要求，[技术特点1]、[技术特点2]。

### [小节标题]
[同上结构]
```

### 简洁明了型应用模板
```markdown
## [章节标题]

### [小节标题]
**关键点**：[关键点1]、[关键点2]。

**数据对比**：
| 项目 | 数据1 | 数据2 | 数据3 |
|-----|-------|-------|-------|
| 项目1 | [数值] | [数值] | [数值] |
| 项目2 | [数值] | [数值] | [数值] |

**结论**：[直接结论]。

### [小节标题]
[同上结构，保持简洁]
```

### 详实丰富型应用模板
```markdown
## [章节标题]

### [小节标题]
[引言，背景说明]。

**历史发展**：[历史发展过程]。[历史阶段1]、[历史阶段2]、[历史阶段3]。

**国内外对比**：
- 国内情况：[国内发展现状、数据]
- 国外情况：[国外发展现状、数据]
- 对比分析：[对比结论]

**典型案例**：[案例1]。[案例2]。[案例分析]。

**数据支撑**：
1. [数据1]：[数据说明，来源XXX]
2. [数据2]：[数据说明，来源YYY]
3. [数据3]：[数据说明，来源ZZZ]

**综合分析**：[多角度分析]。[结论1]、[结论2]。[最终结论]。

### [小节标题]
[同上结构，保持详实]
```

### 图文并茂型应用模板
```markdown
## [章节标题]

### [小节标题]

[图表1]

图表1说明：[图表详细说明]。

[图表2]

图表2说明：[图表详细说明]。

**数据总结**：[关键数据总结]。

### [小节标题]
[同上结构，以图表为中心]
```
